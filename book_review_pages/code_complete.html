'Code Complete' by Steve McConnell

    Chapter 1:
  • Detailed design -> coding -> debugging -> integration -> developer testing (unit/integration).
  • Plan to throw away.

    Chapter 2:
  • Appeal to logic, analogy, and data.
  • Foundation is problem definition.
  • Remove assumptions. Have clearly written requirements/design/etc (checklist). That said, know they will evolve(ch1).
  • Make sure that every decision is explained clearly.
  • Error processing is a beast on its own. Don't forget about 'Fault Tolerance'. (checklist)

    Chapter 4:
  • Understand the distinction between 'programming in a language' (#) vs 'programming into a language' (*). In #, foundation is the language. In *, it is the problem definition.

    Chapter 5:
  • Design is nothing but tradeoffs and priorities.
  • Things have essential and accidental properties. Car's engine (should be paid more attention) vs its shiny logo. Software is no different.
  • Simplify as much as possible. Not more. Software -> subsystems/packages -> classes
  • Use design patterns/std.lib as much as possible.
